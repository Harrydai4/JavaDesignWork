## 一、目标与验收口径

### 1.1 最终交付物（课程层面）

- **程序源码**（ZIP）：能解压后在说明下编译运行。  
- **综合实习论文 PDF**：含分析、设计、实现要点、测试、界面截图、个人总结（格式见 `附件3`）。  
- **答辩**：演示功能并回答问题（成绩占比较高）。

### 1.2 功能验收清单（与题目「基本功能必须全部实现」对齐）

开发过程中建议打印成表，**完成一条勾一条**：

| # | 功能 | 验收要点 |
|---|------|----------|
| F1 | 联系人增删改 | 字段完整录入；删后列表与文件一致 |
| F2 | 分组增删 | 删组**不删联系人**，仅解除归属 |
| F3 | 分组调整 | 多选联系人 ↔ 多分组；支持从组内移除 |
| F4 | 按分组查询 | 「全部」与某组下列表正确；可配置**表格列显示字段** |
| F5 | 搜索 | 支持姓名/电话/手机模糊；支持姓名**全拼/声母**（如 `xl`）；多结果**按姓名排序**并按**姓或姓的拼音首字母**分类展示 |
| F6 | 导出 | **CSV**、**vCard**；可选全部或所选 |
| F7 | 导入 | **CSV**、**vCard**，**追加**导入；与手机样例互测 |
| F8 | 数据持久化 | 退出再启动数据仍在；**文件系统**存储 |
| F9 | 主界面 | **左侧分组** + **右侧联系人列表**（题目图示为参考） |

### 1.3 数据字段验收（每个联系人至少具备）

姓名、电话、手机、即时通信工具及号码、电子邮箱、个人主页、生日、**像片**、工作单位、家庭地址、邮编、**所属组**、备注。

> 「所属组」可与内部分组模型配合：多对多（一人多组）更贴合「把一个或多个联系人放入一个或多个分组」。

## 二、技术选型建议（组内必须统一）

### 2.1 GUI：Swing 还是 JavaFX？

| 维度 | Swing | JavaFX |
|------|-------|--------|
| 资料与教材匹配度 | 高 | 中高 |
| 主界面实现 | `JTree` + `JTable` 常见 | `TreeView` + `TableView` |
| 课程答辩讲解 | 通常更容易举例 | 需讲清 Scene/FXML 等 |

**建议**：组内无人熟悉 GUI 时优先 **Swing**；若追求界面观感且愿意学，可选 **JavaFX**。  
**选定后全文档按一种写法理解**（下文示例多以 Swing 名词描述，JavaFX 可一一对应）。

### 2.2 持久化格式（在「文件存储」前提下）

| 方案 | 优点 | 注意 |
|------|------|------|
| **JSON**（推荐起步） | 易读、易调试；可用 Jackson/Gson | 需在 `pom.xml` 增加依赖 |
| **XML** | 结构化直观 | 手写或 JAXB，代码略长 |
| **纯 Java 序列化** | 实现快 | 版本兼容性差，不利于答辩展示数据文件 |

**建议**：内存模型 + **JSON 文件**（联系人、分组、关系），像片文件放 `data/photos/`，JSON 存相对路径。

### 2.3 拼音搜索

- 使用 **pinyin4j**（或同类库）生成 **全拼**、**首字母串**，在建立/修改联系人时缓存到领域对象或索引结构中，避免每次搜索全表转拼音。  
- 题目要求「声母」匹配：可将姓名转为**首字母序列**（如「张三」→ `zs`），搜索词 `z`/`zs`/`zhang` 等规则在组内写清楚并**写入论文**。

### 2.4 依赖与课程规定

- 使用 Maven 管理依赖前，**确认课程是否允许第三方库**；若限制严格，CSV/拼音可手写简化版，但工作量与风险需自行评估。

## 三、推荐总体架构

### 3.1 逻辑分层

```
ui          → 仅负责界面与事件，调用应用服务
service     → 业务规则（分组约束、搜索排序、导入合并策略）
model       → Contact、Group、分组关系等纯数据与校验
storage     → 读写磁盘（JSON/XML/vCard/CSV 等）
importexport→ vCard、CSV 专用解析与生成（可并入 storage）
util        → 拼音、路径、日期格式化
```

### 3.2 建议包结构（在 `com.javadesign.addressbook` 下）

```
com.javadesign.addressbook
├── AddressBookApp.java          # 入口：启动主窗口
├── model/
│   ├── Contact.java
│   ├── Group.java
│   └── ...                      # 如 GroupMembership 多对多
├── service/
│   ├── AddressBookService.java  # 门面，协调各服务
│   ├── ContactService.java
│   ├── GroupService.java
│   └── SearchService.java
├── storage/
│   ├── AddressBookRepository.java  # 接口
│   └── JsonAddressBookRepository.java
├── importexport/
│   ├── CsvHandler.java
│   └── VCardHandler.java
└── ui/
    ├── MainFrame.java
    └── ...                      # 编辑对话框、列选择等
```

### 3.3 数据关系（建议）

- **Contact**：唯一 ID（UUID 或自增 long），含全部题目字段 + 照片路径。  
- **Group**：唯一 ID + 名称。  
- **联系人与分组**：多对多用 `Set<String> groupIds` 存在 Contact 内，或单独 `contact_group` 映射表结构（内存中用 `Map`/`集合` 表达即可）。

## 四、界面设计要点（主界面 + 子对话框）

### 4.1 主窗口布局（Swing 示例）

- **`BorderLayout`**：  
  - **North**：菜单栏（文件：导入/导出/退出）+ 搜索框 + 搜索按钮。  
  - **West**：`JScrollPane` + `JTree` 显示分组树；含节点「全部联系人」+ 各分组。  
  - **Center**：`JTable` 显示联系人列表；表头可动态列（对应 F4）。  
  - **South**：状态栏（可选）：当前选中分组、记录数。

### 4.2 必要子界面

- **联系人编辑对话框**：`JDialog`，分栏或选项卡录入全部字段；像片用「选择文件」复制到 `data/photos` 并保存路径。  
- **分组管理**：新增分组输入框；删除分组前 **二次确认**，逻辑为「仅删组、联系人改归属」。  
- **列显示设置**：复选框列表或多选列表，决定 `JTable` 哪些列可见（持久化到本地 preferences 或 JSON 配置）。

### 4.3 搜索与结果展示

- 搜索框独立于分组树：过滤后可在主表展示；若题目要求「按姓拼音首字母分类」，可在表上方用 `JList`/`JTable` 分组渲染，或**分区块**展示（实现方式写进论文）。

## 五、分阶段实施计划（建议里程碑）

### 阶段 M1：领域模型与内存数据（1～2 天）

- 实现 `Contact`、`Group` 与校验（邮箱、电话简单格式可选）。  
- `AddressBookService` 内存中 `List`/`Map` 管理，**暂不写文件**。  
- 简单控制台或最小窗口验证增删改。

### 阶段 M2：持久化（2～3 天）

- 定义 `AddressBookRepository`；实现 JSON 读写（或 XML）。  
- 启动加载、退出保存；异常处理与用户提示（` JOptionPane`）。  
- 数据目录：建议 `用户目录/.addressbook/` 或项目下 `data/`（答辩时说明路径策略）。

### 阶段 M3：主界面骨架（2～4 天）

- `MainFrame`：左侧树 + 右侧表；树选中切换过滤列表。  
- 表格列与模型 `TableModel` 绑定 `Contact` 列表。

### 阶段 M4：分组与多选逻辑（2～3 天）

- 分组增删（删组不删人）。  
- 多选联系人（`JTable` 行选择）→ 加入组 / 移出组。

### 阶段 M5：搜索与拼音（3～5 天）

- 接入 pinyin4j；缓存拼音字段。  
- 实现模糊规则与排序、分类展示。

### 阶段 M6：导入导出（3～5 天）

- CSV：定义列顺序与转义规则；与 Excel 互开测试。  
- vCard：至少支持 `FN`、`TEL`、`EMAIL`、`BDAY`；像片 `PHOTO` 与手机导出样例对照调试。

### 阶段 M7：测试、打包、论文素材（贯穿后期）

- 按附件3写**测试用例表**（输入、预期、实际）。  
- 截图主界面、导入导出、搜索。  
- `mvn package` 可执行 jar（可选 `maven-assembly-plugin` 或 `javafx-maven-plugin` 视选型而定）。

> 时间为粗略估计，按组人数与基础浮动。

## 八、风险与对策

| 风险 | 对策 |
|------|------|
| vCard 与手机格式不一致 | 早用真实手机导出 `.vcf` 做黄金样例；导入做宽松解析 |
| 拼音多音字 | 在论文中说明采用「默认读音」或仅支持常用名 |
| 大图片内存 | 列表用缩略图或仅路径，打开详情再加载原图 |
| 合并答辩时间紧 | 按 M1→M7 保核心：先持久化+主界面+CRUD，再搜索与导入导出 |
