下面是你选的**题目3：通讯录管理程序**中与开发直接相关的硬性要求（答辩前建议逐条能演示）。

#### 数据与存储

- **必须用文件系统存储**（文本/XML/vCard/对象序列化/自定义格式等均可）。  
- **每个联系人至少包含字段**：姓名、电话、手机、即时通信工具及号码、电子邮箱、个人主页、生日、**像片**、工作单位、家庭地址、邮编、**所属组**、备注。

#### 界面（Swing 或 JavaFX）

- **主界面**需包含：  
  - **左侧**：联系人**分组**资料；  
  - **右侧**：联系人**列表**。

#### 功能（题目声明「基本功能必须全部实现」）

1. **联系人管理**：增加、修改、删除。  
2. **联系组管理**：增加分组；删除分组时**不得删除联系人**，而是使联系人**不再属于该组**。  
3. **联系人分组调整**：把一个或多个联系人放入一个或多个分组；从某分组中移除。  
4. **查询**：在「全部」或某分组下列出联系人；并支持**调整列表中显示的联系人属性**（列显示哪些字段）。  
5. **搜索**（模糊查询）：在搜索框输入以下内容之一，列出符合条件列表：  
   - 姓名、电话、手机；  
   - **姓名的汉语拼音全拼或声母**（例：输入 `xl` 匹配声母为这两字母的条目）。  
   - 多结果时：**按姓名排序**，并按**姓**或**姓的拼音首字母**分类展示（题目明确要求）。  
6. **导入/导出**：  
   - 导出：**CSV**、**vCard**（全部或部分内容）；  
   - 导入：CSV、vCard，**追加**到当前通讯录；  
   - vCard 需尽量与**主流手机/通讯录软件**兼容。

## 3. 新人需要掌握的知识地图（按「题目3」裁剪）

不必一次学完，建议按下面顺序：**能跑 Hello World → 能写类与集合 → 能做 Swing 界面 → 再攻文件与 vCard/CSV → 最后拼音搜索**。

### 3.1 Java 语言（最低限度）

- **类、对象、构造方法、getter/setter**；`String` 常用方法。  
- **集合**：`List`、`Map`（存联系人列表、分组到联系人映射）。  
- **异常**：`try-catch-finally`，文件读写必遇 `IOException`。  
- **接口与实现分离**：例如「存储接口」+「文件实现」，方便以后改格式。  
- **日期**：生日可用 `LocalDate`（Java 8+），注意解析与格式化。

### 3.2 桌面界面（二选一，团队统一）

- **Swing**：`JFrame`、`JPanel`、`JTree`/`JList`/`JTable`、`JFileChooser`、布局 `BorderLayout` 等。  
- **JavaFX**：`Stage`、`Scene`、`TreeView`、`TableView`、`FileChooser`。  

建议：若组内没人做过界面，选 **Swing 资料更多、示例更多**；若追求现代感可选 JavaFX。

### 3.3 文件与数据格式（本题核心）

- **文本/对象文件**：`Files`、`BufferedReader`，或对象序列化（需评估版本兼容性）。  
- **XML**：可选 `DOM`/`JAXB` 或第三方库，用于自定义格式。  
- **CSV**：手写解析注意引号与逗号；或用小库（团队需确认课程是否允许第三方依赖）。  
- **vCard（.vcf）**：需了解 **VCF 3.0/4.0** 常见字段（`FN`、`TEL`、`EMAIL`、`BDAY`、`PHOTO` 等）；导入导出要与手机导出的样例文件**互测**。  
- **照片**：题目要求「像片」字段——通常存**磁盘路径**或**把图片复制到应用数据目录**并在数据文件里记相对路径；不要把几 MB 的 Base64 硬塞进一个超小文本里导致性能差（可写进 vCard 的 `PHOTO;ENCODING=b` 需权衡）。

### 3.4 拼音与搜索（难点，提前预留时间）

- **汉字转拼音**：可使用开源库（如 **pinyin4j** 等）生成全拼与首字母，用于声母/全拼搜索。  
- **模糊匹配**：`String.contains`、或对电话只保留数字再匹配。  
- **排序与分组**：`Comparator` 按姓名排序；「按姓分类」可先取首字符（或姓的拼音首字母）作为分组键。

### 3.5 构建与工程化（强烈建议）

- **JDK**：建议 **17 或 21（LTS）**。  
- **Maven 或 Gradle**：管理依赖（拼音库、可能的 CSV/vCard 库）。  
- **Git**：至少会 `commit`、分支简单协作；提交 ZIP 前清理 `target/` 等构建产物（按老师要求）。

## 4. 环境配置：从零到「能编译运行 Java」— 逐步操作

以下以 **Windows 10/11**、**JDK 17+**、**Maven**、**IntelliJ IDEA** 为例（可替换为 VS Code + Java 插件，但课堂与答疑通常 IDEA 更方便）。

### 4.1 安装 JDK

1. 从 **Adoptium（Eclipse Temurin）** 或 **Oracle JDK** 下载 **Windows x64** 的 **JDK 17 或 21** 安装包。  
2. 安装后记下安装路径，例如：`C:\Program Files\Eclipse Adoptium\jdk-17.x.x-hotspot\`。  
3. 配置环境变量：  
   - **新建**系统变量 `JAVA_HOME` = 上述 JDK 根目录。  
   - 在 **Path** 中新增：`%JAVA_HOME%\bin`。  
4. **新开**一个 PowerShell 或 CMD，执行：

```bat
java -version
javac -version
```

两行都应显示版本号且**无**「不是内部或外部命令」。

### 4.2 安装 Maven（若用 Maven 工程）

1. 从 Apache Maven 官网下载二进制 zip，解压到如 `C:\tools\apache-maven-3.9.x\`。  
2. 新建 `MAVEN_HOME` = 解压目录；Path 增加 `%MAVEN_HOME%\bin`。  
3. 新开终端执行：

```bat
mvn -v
```

应显示 Maven 版本与 Java 版本。

### 4.3 安装 IDE

1. 安装 **IntelliJ IDEA Community**（免费版即可）。  
2. 首次启动：勾选关联 `.java`，可选安装自带 Maven。  
3. 用 IDE 新建一个 **Maven** 项目，模板选 `maven-archetype-quickstart` 或空 Maven 项目，运行 `main` 打印一行字，确认**能运行**。

### 4.4 建议的自检清单（开发前打勾）

- [ ] `java -version` / `javac -version` 正常  
- [ ] IDE 能新建并运行 Java 程序  
- [ ] Maven（若使用）`mvn -v` 正常  
- [ ] 全组 **JDK 主版本一致**（避免一人 8 一人 21 导致语法/类库问题）

## 5. 推荐的项目架构（单机 Swing/JavaFX + 文件存储）

不必一开始完美，但建议**早期**定好分层，后期加导入导出、搜索时不乱。

```
ui（界面）     → 只负责显示与事件，调用应用服务
service（业务）→ 校验、分组逻辑、搜索排序规则
model（领域）  → Contact、Group 等实体
storage（持久化）→ 读写文件：CSV/XML/vCard/图片路径等
util（工具）   → 拼音、字符串规范化（电话只保留数字）等
```

**数据文件建议**：  
- 自定义目录：`data/contacts.json` 或 `contacts.xml` + `photos/` 存图片；  
- 导入导出时再与 **CSV/vCard** 互转。

## 6. 开发顺序建议（降低返工）

1. **模型**：`Contact`、`Group` 字段与校验（手机、邮箱简单格式）。  
2. **内存版**：增删改查 + 列表展示（无文件）。  
3. **持久化**：保存到文件，重启后能加载。  
4. **界面**：左侧分组树 + 右侧表格；完成分组增删与「删组不删人」。  
5. **搜索**：先字符串包含；再加拼音。  
6. **导入导出**：先做 CSV，再做 vCard；用真实手机导出文件测兼容性。  
7. **论文与测试**：按附件3写测试用例表与截图。
